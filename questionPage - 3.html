    • python conditional debug breakpoint one-liner for versions before 3.7 PEP 553 that acts similarly to Perl's $DB::single=1

    In python versions before PEP 553 breakpoint() utility, what is the recommended way to add (ideally a
    one-liner) code to have a breakpoint that can be ignored upon a condition (e.g. a global debug flag or
    args.debug flag).

    If I run this code as: perl code.pl, it will run to completion. If I run this code with: perl -d
    code.pl, the pdb will stop at the breakpoint line (not before the next line with a my $b = 2;
    statement) because it contains a 1; statement after the $DB::single=1; statement;

    Asked by: Millheight Anthony

    2 Answers

    Same as perl, python can be run with -d to set a debug flag:

    $ python --help
    [...]
    -d : debug output from parser; also PYTHONDEBUG=x
    [...]

    it gets a little trickier though, since python doesn't allows mutation of the flags structure, or even creating an instance of it:

    >>> import sys
    >>> sys.flags.debug = 0 # no mutating ...
    Traceback (most recent call last):
    File "stdin", line 1, in module
    TypeError: readonly attribute
    >>> type(sys.flags)() # ... and no instanciating
    Traceback (most recent call last):
    File "input", line 1, in module
    TypeError: cannot create 'sys.flags' instances
